# Task: Project scaffolding — TypeScript CLI with config loading

## Description
Initialize the ai-cartographer npm package with TypeScript, configure the CLI framework using commander, and set up config loading with cosmiconfig. This is the foundation everything else builds on.

## Acceptance Criteria
- [ ] npm package initialized with proper package.json (name: ai-cartographer, bin field configured)
- [ ] TypeScript configured with tsconfig.json (strict mode, ES2020 target)
- [ ] Commander CLI wired with subcommands: init, refresh, status, config
- [ ] Global flags registered: --verbose, --free, --dry-run, --output, --format
- [ ] Cosmiconfig loads `.ai/cartographer.config.json` with sensible defaults
- [ ] Default config includes: ignorePaths, keyEntryPoints, outputPath, llm provider/model settings
- [ ] `npx ai-cartographer --help` displays all commands and options
- [ ] Build script produces working CLI binary

## Technical Details
- Use commander for CLI parsing
- Use cosmiconfig for config loading (supports .json, .yaml, .js)
- Default ignore patterns: node_modules, .git, dist, build, vendor, __pycache__, binary extensions
- Default output: .ai/context-map.md
- Structure: src/cli.ts (entry), src/commands/*.ts, src/config.ts
